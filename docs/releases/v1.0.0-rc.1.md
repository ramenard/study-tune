# v1.0.0-rc.1

Durcissement de la certification Bloc 2 : sécurité, RGPD, tests, accessibilité, déploiement.

## Sécurité
- En-têtes HTTP Helmet et limitation de débit (100 req/min global, 5/min sur login/register).
- Secret partagé exigé sur le webhook Kie (`/api/music/webhook/kie/:secret`).
- Tokens JWT courts (15 min) + refresh tokens rotatifs (7 j, hash bcrypt) + rafraîchissement automatique client sur 401.

## RGPD
- Suppression de compte (`DELETE /api/auth/me`) avec cascade (musiques + objets MinIO, playlists, amitiés).
- Consentement obligatoire à l'inscription (`consentAt`).

## Tests & CI
- Seuils de couverture Jest + tests guard/strategy/controller.
- Tests end-to-end réels du flux d'authentification (PostgreSQL).
- Couverture Vitest + specs de composants `generate` / `library`.
- CI : couverture en artifact, e2e et `npm audit`.

## Accessibilité
- Contrôles clavier, `aria-label`, labels de formulaires, `aria-live`, `lang="fr"`, `:focus-visible`, règles ESLint a11y.

## Infrastructure
- Image Docker client (nginx, fallback SPA, proxy `/api`), profil compose `full`, `restart: unless-stopped`.

## Nettoyage
- Suppression des modules `chat` et `gemini` inutilisés.
